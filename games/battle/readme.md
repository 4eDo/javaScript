# Описание структуры предметов

Файлы: `items.json`, `itemsUpgrade.json`

---

## Общая структура

```json
{
  "id": "уникальный_идентификатор",
  "name": "Название предмета",
  "descr": "Описание предмета",
  "level": 1,
  "tags": ["массив", "тегов"],
  "slots": ["массив", "слотов"],
  "properties": { },
  "usage": { },
  "img": "URL картинки"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | строка | Уникальный идентификатор предмета. Используется в рецептах, инвентаре, экипировке |
| `name` | строка | Отображаемое название |
| `descr` | строка | Текстовое описание (опционально) |
| `level` | число | Уровень предмета. Влияет на фильтры и шанс дропа с врагов (чем выше, тем ниже шанс) |
| `tags` | массив строк | Теги предмета |
| `slots` | массив строк | В какие слоты можно экипировать |
| `properties` | объект | Пассивные свойства (работают всегда, пока предмет надет) |
| `usage` | объект | Свойства расходника (только для предметов с тегом `consumable` в слоте `consumable`) |
| `img` | строка | URL изображения предмета |

---

## Теги (`tags`)

| Тег | Описание |
|-----|----------|
| `weapon` | Оружие |
| `armor` | Броня |
| `consumable` | Расходник. Может иметь блок `usage` |
| `clothes` | Одежда |
| `material` | Материал для крафта |
| `twoHanded` | Двуручное оружие. Занимает оба слота `weapon-0` и `weapon-1`. Нельзя надеть второе оружие |
| `single` | На одну часть тела можно надеть только один предмет с этим тегом. |

---

## Слоты (`slots`)

| Слот | Часть тела | Макс. слотов |
|------|-----------|-------------|
| `head` | Голова | 3 |
| `body` | Тело | 3 |
| `legs` | Ноги | 3 |
| `weapon` | Оружие | 2 (двуручное занимает оба) |
| `arm_l` | Левая рука | 2 |
| `arm_r` | Правая рука | 2 |
| `feet_l` | Левая ступня | 3 |
| `feet_r` | Правая ступня | 3 |
| `belt` | Пояс | 3 |
| `consumable` | Расходники | Зависит от `CAPACITY` надетых поясов |

---

## Пассивные свойства (`properties`)

Работают всегда, пока предмет надет в любом слоте (кроме `consumable`).

| Ключ | Описание | Влияние |
|------|----------|---------|
| `STR` | Сила | + к урону (1 к 1) |
| `CON` | Стойкость | + к максимальному HP (10 HP за 1 CON) |
| `AGI` | Ловкость | Влияет на шанс уклонения |
| `REG` | Регенерация | Восстановление HP каждый ход врага |
| `ACC` | Меткость | Влияет на шанс попадания |
| `DEF` | Защита | Поглощает урон 1 к 1 |
| `HP` | Здоровье | + к максимальному HP |
| `DAMAGE_MIN` | Мин. урон | Нижняя граница урона оружия (только для `weapon`) |
| `DAMAGE_MAX` | Макс. урон | Верхняя граница урона оружия (только для `weapon`) |
| `CAPACITY` | Вместимость | Добавляет слоты для расходников (только для `belt`) |

**Формулы:**
- HP = 100 + CON × 10
- Урон в бою = случайное (DAMAGE_MIN…DAMAGE_MAX) + STR − DEF врага
- Шанс попадания = 100 − (AGI защитника / ACC атакующего) × 100, минимум 5%
- Шанс уклонения = (AGI / ACC врага) × 100

## Расходник (`usage`)

Работает только когда предмет с тегом `consumable` экипирован в слот `consumable`. После срабатывания удаляется из слота. Один расходник за ход.

```json
"usage": {
  "trigger": { "target": "...", "val": число, "op": "оператор" },
  "duration": число или "battle",
  "properties": { }
}
```

### Триггер (`trigger`)

| Поле | Описание |
|------|----------|
| `target` | Что проверяем |
| `val` | Пороговое значение |
| `op` | Оператор сравнения: `eq` (равно), `lt` (меньше), `gt` (больше), `lte` (≤), `gte` (≥). По умолчанию `eq` |

**Значения `target`:**

| target | Условие срабатывания |
|--------|---------------------|
| `hp` | Текущее HP персонажа |
| `hp_pct` | Текущее HP в процентах от максимума |
| `miss_streak` | Промахи подряд |
| `hit_streak` | Попадания подряд |
| `enemy_count` | Количество живых врагов |
| `enemy_level` | Уровень врага относительно уровня игрока (+val) |
| `on_dodge` | После уклонения игрока |
| `battle_start` | В начале боя |

### Длительность (`duration`)

| Значение | Смысл |
|----------|-------|
| `0` | Мгновенный эффект |
| число | Количество ходов |
| `"battle"` | До конца боя |

### Эффект (`properties`)

Те же ключи, что и в пассивных свойствах. При мгновенном эффекте (`duration: 0`) имеет смысл только `HP` (лечение). При длительном — любые статы.

## Рецепты (`itemsUpgrade.json`)

```json
{
  "result": "id_результирующего_предмета",
  "count": 1,
  "ingredients": [
    { "id": "id_ингредиента", "count": 3 }
  ]
}
```

| Поле | Описание |
|------|----------|
| `result` | ID создаваемого предмета |
| `count` | Сколько создаётся (по умолчанию 1) |
| `ingredients` | Массив ингредиентов |
| `ingredients.id` | ID ингредиента |
| `ingredients.count` | Требуемое количество |

Ингредиенты тратятся сначала из инвентаря, потом из экипировки. Если ингредиент надет — показывается предупреждение.

## Враги (`enemy.json`)

```json
{
  "id": "zombie_basic",
  "name": "Зомби",
  "baseStats": { },
  "levelUps": { },
  "equipSlots": ["head", "body"],
  "dropChance": 0.15
}
```

| Поле | Описание |
|------|----------|
| `baseStats` | Базовые статы на уровне 1 |
| `levelUps` | Прирост статов в % за каждый уровень сверх 1 |
| `equipSlots` | В какие слоты враг может надеть экипировку (кроме `weapon`) |
| `dropChance` | Базовый шанс дропа вещи с врага |

**Статы врага:** HP, STR, CON, AGI, REG, ACC, DEF, DAMAGE_MIN, DAMAGE_MAX, xp.

**Расчёт статов от уровня:** `стат = baseStat + baseStat × (levelUp / 100) × (уровень − 1)`.

**Дроп:** шанс = `dropChance × (1 − level × 0.1)`, минимум 2%.
